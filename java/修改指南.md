## 📋 项目基本信息

### 项目描述
- **队伍**：FTC32477
- **赛季**：Decode（2025-2026 赛季）
- **程序类型**：远程遥控(TeleOp)控制程序
- **主要功能**：
  - 麦克纳姆轮底盘全向移动（前后、左右、旋转）
  - 拾取系统（正向、反向）
  - 装填系统
  - 发射系统（开环功率控制，0.0-1.0，步长0.05）

### 赛季主题背景
**Decode 赛季**要求机器人能够：
1. 拾取游戏物体
2. 运输和装填
3. 发射到目标区域

本程序为完整的远程遥控解决方案。

## ⚠️ 关键修改原则

### 原则 1：先修改分布式文件，再修改集中式文件

**分布式（多文件）版本**：
```
TeleOp_All_v2.1/ (推荐)
├── TeleOp_2_1.java
├── ChassisDriveSystem_2_1.java
├── SubsystemManager_2_1.java (独立控制)
├── ControlInputManager_2_1.java (v1.0按键)
├── RobotConstants_2_1.java (±200精度)
├── NavigationSystem_2_1.java
└── TelemetryManager_2_1.java
```

**集中式（单文件）版本**：
```
TeleOp_All_v2.1/
└── TeleOp_All_2_1.java (内部8个类，推荐竞赛部署)
```

**修改流程**：
1. ✅ 修改多文件版本的相应源文件
2. ✅ 验证多文件版本编译无误
3. ✅ 将相同改动集成到单文件版本的对应内部类
4. ✅ 验证单文件版本编译无误

### 原则 2：版本号一致性

所有文件和类必须遵循版本号约定：
- **v1.0** 系列：所有文件/类名后缀为 `_1_0`
- **v2.0** 系列：所有文件/类名后缀为 `_2_0`
- **v2.1** 系列：所有文件/类名后缀为 `_2_1`（v2.0 优化版本）

**强制规则**：
- 类名必须与文件名中的版本号相匹配
- 所有相互引用的类名必须一致
- 构造函数名必须与类名相同

### 原则 3：独立性原则

v1.0、v2.0、v2.1 完全独立：
- ❌ 不允许跨版本引用类（如 v1.0 引用 v2.0 或 v2.1）
- ❌ 不允许混用版本常量
- ✅ 不同版本可有相同的功能但独立实现
- ⚠️ v2.1 基于 v2.0 优化，可参考 v2.0 原理但独立实现

### 原则 4：v2.1 特殊性

v2.1 是 v2.0 的优化版本，有三个核心改动：
1. **独立控制** - SubsystemManager_2_1 三个系统完全分离（无联动）
2. **统一精度** - RobotConstants_2_1 所有档位 ±200 RPM
3. **v1.0 按键** - ControlInputManager_2_1 恢复熟悉的按键映射

## 🔄 常见修改场景

### 场景 1：修改底盘运动公式

**涉及文件**：
- 多文件：`ChassisDriveSystem_1_0.java` / `ChassisDriveSystem_2_0.java`
- 单文件：`TeleOp_All_1_0.java` / `TeleOp_All_2_0.java` 中的 `ChassisDrive` 内部类

**修改步骤**：
1. 打开多文件版本的 `ChassisDriveSystem_1_0.java`
2. 找到 `calculateWheelPowers()` 方法
3. 修改麦克纳姆轮公式的计算逻辑
4. 验证编译无误
5. 在单文件版 `TeleOp_All_1_0.java` 中找到 `ChassisDrive` 内部类
6. 找到对应的功率计算逻辑（搜索"麦克纳姆轮公式"或"powerXX = drive ..."）
7. 应用同样的修改
8. 验证编译无误

**例子**：修复麦克纳姆轮平移和旋转的 strafe 符号问题
```java
// 修改前（错误）
double powerBL = inputDriveFB - inputStrafeLR + inputRotateCW;
double powerBR = inputDriveFB + inputStrafeLR - inputRotateCW;

// 修改后（正确）
double powerBL = inputDriveFB + inputStrafeLR - inputRotateCW;
double powerBR = inputDriveFB - inputStrafeLR + inputRotateCW;
```

## 🚫 禁止项和风险操作

### ❌ 禁止操作
1. **混合两个版本的代码**
   - 不要在 v1.0 中使用 v2.0 的类
   - 不要改用 `_1_0_0` 和 `_2_0_0` 混合的后缀

2. **不一致的版本号**
   - 所有相关文件和类必须使用同一个版本号后缀
   - 例：如果用 `_1_0`，所有 v1.0 文件都应该是 `_1_0`

3. **忘记同步修改**
   - 修改多文件版本后必须同时修改单文件版本
   - 修改单文件版本后需要反向同步到多文件版本（如有必要）

4. **改动硬件映射名称后忘记验证**
   - 修改 `CHASSIS_MOTOR_FRONT_LEFT_NAME` 等常量后必须确保对应的硬件有该名称

### ⚠️ 风险操作
1. **大规模重构**
   - 建议一次只修改一个功能模块
   - 修改后立即编译测试

2. **删除"似乎未使用"的代码**
   - 某些代码可能在特定模式下使用
   - 删除前先搜索所有引用

3. **修改基础的数学公式**
   - 底盘控制、PID、坐标变换等
   - 修改前充分理解原理，修改后充分测试
